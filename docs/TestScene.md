# 测试场景设计规范

> 本文档定义用于后视镜视野仿真的 4 种标准化测试场景的设计依据、空间布局和使用方法。
>
> **配置文件**：`config/scenarios.yaml`、`config/test_scene.yaml`
> **代码实现**：`src/scene_builder.py`、`src/scenarios/`、`src/objects/`
> **预构建文件**：`assets/blender-scenes/scene_*.blend`

---

## 一、设计原则

1. **真实比例**：所有尺寸以米为单位，符合真实道路几何标准
2. **参数化**：道路宽度、车道数、参考物间距等均为可调参数（`config/` 目录）
3. **功能导向**：场景元素服务于视野仿真分析，不追求视觉精美
4. **轻量化**：参考车辆用方块、交通锥用锥体，避免高面数模型影响渲染
5. **坐标系一致**：全项目统一坐标系（见下文）

---

## 二、坐标系与空间范围

### 2.1 坐标系约定

| 方向 | 轴 | 说明 |
|------|------|------|
| 车头 | **+Y** | FORWARD |
| 车尾 | **-Y** | REAR |
| 驾驶员侧 | **-X** | LEFT（左舵车）|
| 副驾驶侧 | **+X** | RIGHT |
| 向上 | **+Z** | UP |

**原点**：前轴中心地面投影 `(0, 0, 0)`

### 2.2 场景空间范围

| 轴 | 范围 | 设计依据 |
|----|------|---------|
| X | -15m ~ +15m | 覆盖多车道 + 路肩 |
| Y | -80m ~ +30m | 后方 80m 覆盖 R46 法规全范围 + 大型房车长度；前方 30m 提供道路透视感 |
| Z | 0 ~ 5m | 覆盖房车高度 + 参考柱 |

## 五、场景 1：高速公路行驶

### 5.1 概述

模拟车辆+房车组合在 3 车道高速公路中央车道行驶的真实场景。左右车道后方各放置参考车辆，用于验证驾驶员通过后视镜能否观察到邻车道的交通状况。

### 5.2 道路布局

使用 `highway_3lane` 预设（单向 3 车道）：

```
                ← 7.5m →
    路肩 | 车道3 | 车道2 | 车道1 | 分隔带 | 车道1 | 车道2 | 车道3 | 路肩
          慢车道   本车道   快车道   1.5m    ← 对向车道 →
              +3.75   0.00   -3.75
```

车辆原点 (0,0,0) 位于本侧车道 2 中央。各车道 X 坐标（相对于车辆）：

| 车道 | X 坐标 | 说明 |
|------|--------|------|
| 车道 1（快车道）| -3.75m | 左侧 |
| 车道 2（本车道）| 0.00m | 车辆所在 |
| 车道 3（慢车道）| +3.75m | 右侧 |

### 5.3 参考车辆

参考车辆使用简化的彩色长方体（4.5m × 1.85m × 1.5m），底部在地面上。

| 车辆 | 车道 | X | Y（后方距离）| 颜色 | 用途 |
|------|------|---|-------------|------|------|
| 左后近车 | 快车道 | -3.75 | -15m | 蓝色 | 近距离左后方交通 |
| 左后远车 | 快车道 | -3.75 | -35m | 浅蓝 | 远距离左后方交通 |
| 右后近车 | 慢车道 | +3.75 | -12m | 绿色 | 近距离右后方交通 |
| 右后远车 | 慢车道 | +3.75 | -40m | 浅绿 | 远距离右后方交通 |

### 5.4 附加元素

- **中央护栏**：灰色金属护栏（H=0.8m），位于中央分隔带两侧
- **道路标线**：完整的车道虚线、边缘实线、中央双黄线

### 5.5 验证方法

1. 导入车辆+房车+后视镜模型
2. 从驾驶员眼点渲染后视镜视角
3. 检查 4 辆参考车是否在左/右后视镜中可见
4. 对比不同后视镜类型（标准 vs 加长拖车镜）的覆盖差异

## 六、场景 2：停车场静态测试

### 6.1 概述

车辆+房车组合静止在开阔的停车场中，周围按网格排列交通锥和立杆。通过渲染后视镜视角，系统化地扫描后方每个方位和距离的可见性，生成"可视范围热力图"。

### 6.2 地面

4. 对比不同车辆×房车×后视镜组合的覆盖差异

## 七、场景 3：UN ECE R46 法规验证

### 7.1 概述

根据 UN ECE Regulation No. 46（间接视野装置法规）中 Class III 后视镜的视野要求，在地面标记法规规定的最小可视区域，并在关键边界位置放置测试柱。通过渲染后视镜视角，直接判断是否满足法规要求。

### 7.2 法规依据

**UN ECE R46，第 15.2.4.3.2 段**（Class III 外后视镜）：

> 驾驶员必须能通过后视镜看到：
> - **远场**：从驾驶员眼点后方 **20m** 到地平线，一条宽度至少 **4m** 的地面条带，其内侧边界为平行于车辆纵向中面、经过车辆（或拖挂组合）最外侧点的平面
> - **近场**：从驾驶员眼点后方 **4m** 到 **20m**，一条宽度至少 **1m** 的地面条带，同样从最外侧点起算
>
> 驾驶员侧和副驾驶侧各自独立要求。

参考文献：[UN ECE R46 Rev.6 (PDF)](https://unece.org/DAM/trans/main/wp29/wp29regs/2016/R046r6e.pdf)

### 7.3 关键坐标计算

法规区域的坐标取决于具体的车辆+房车组合：

```
X_outer = max(车辆半宽, 房车半宽)
Y_eye   = 驾驶员眼点的 Y 坐标
```

| 区域 | X 范围（副驾驶侧 +X）| Y 范围 | 宽度 |
|------|----------------------|--------|------|
| 近场 | [X_outer, X_outer + 1.0] | [Y_eye - 20, Y_eye - 4] | 1m |
| 远场 | [X_outer, X_outer + 4.0] | [Y_eye - 80, Y_eye - 20] | 4m |

驾驶员侧 (-X) 为镜像对称。

### 7.4 示例：SUV (CR-V) + 大型房车 (L)

- 车辆半宽 = 1.866 / 2 = 0.933m
- 房车半宽 = 2.50 / 2 = 1.25m
- **X_outer = 1.25m**（房车更宽，取较大值）
- **Y_eye = -0.10m**

| 区域 | 颜色标记 | X 范围 | Y 范围 |
|------|---------|--------|--------|
| 副驾近场 | 半透明绿 | 1.25 ~ 2.25 | -4.10 ~ -20.10 |
| 副驾远场 | 半透明蓝 | 1.25 ~ 5.25 | -20.10 ~ -80.10 |
| 驾驶近场 | 半透明黄 | -2.25 ~ -1.25 | -4.10 ~ -20.10 |
| 驾驶远场 | 半透明红 | -5.25 ~ -1.25 | -20.10 ~ -80.10 |

俯视示意图：

```
        驾驶员侧 (-X)                 车辆                副驾驶侧 (+X)
                                  ┌─────────┐
                                  │  车辆    │
                                  │ (0,0,0) │
    ──────────────────────────────┴─────────┴──────────────────────────
Y=-4.1   ┌──────┐                                    ┌──────┐
         │ 近场 │  1m宽                          1m宽 │ 近场 │
         │ 黄色 │                                    │ 绿色 │
Y=-20.1  ├──────┴───────────┐          ┌─────────────┴──────┤
         │    远场          │          │         远场       │
         │    红色   4m宽   │          │   4m宽   蓝色      │
         │                  │          │                    │
Y=-80.1  └──────────────────┘          └────────────────────┘
       -5.25  -1.25                  1.25              5.25
```

### 7.5 场景元素

| 元素 | 类型 | 数量 | 说明 |
|------|------|------|------|
| 可视区域标记 | 半透明地面矩形 | 4 个 | 近场×2 + 远场×2 |
| 最外侧边界线 | 橙色地面线条 | 2 条 | 标记 X = ±X_outer |
| 测试柱 | 白色圆柱 (H=1m) | 10 根 | 每侧 5 根，位于区域角点和中点 |

测试柱位置（副驾驶侧，驾驶侧为 X 镜像）：

| 柱 ID | X | Y | 含义 |
|-------|---|---|------|
| P1 | X_outer + 0.5 | Y_eye - 20 | 远场内边界 20m 处 |
| P2 | X_outer + 4.0 | Y_eye - 20 | 远场外边界 20m 处 |
| P3 | X_outer + 0.5 | Y_eye - 4 | 近场中点 4m 处 |
| P4 | X_outer + 1.0 | Y_eye - 4 | 近场外边界 4m 处 |
| P5 | X_outer + 2.0 | Y_eye - 40 | 远场中点 40m 处 |

### 7.6 验证方法

1. 导入车辆+房车+后视镜模型
2. 渲染后视镜视角图像
3. **合规判断**：P1-P5 测试柱全部在镜中可见 → 满足法规；有不可见 → 不合规
4. 房车本身作为物理遮挡物，无需额外建模——它就是遮挡源
5. 最差情况组合：Polo (最窄车) + 大型房车 L (最宽)，单侧超出 0.37m

### 7.7 为什么需要拖车加长镜

当房车宽度超过牵引车宽度时，标准后视镜（Class III）的视线被房车车身完全遮挡。此时：
- 标准平面镜：无法看到法规要求区域的任何部分
- 标准凸面镜：可能看到极窄的边缘区域，但不满足法规
- **拖车加长镜**：臂长从 0.15m 增加到 0.25m，镜面从 170mm 增加到 200mm，能恢复法规要求的视野覆盖

这正是本仿真系统要量化验证的核心问题。

## 八、场景 4：变道检测

### 8.1 概述

测试后方接近车辆在不同距离下的镜中可见性。与场景 1 的"快照"式布局不同，本场景是"扫描"式——同一辆车在 5 个距离依次出现，用于生成"检测距离-可见性"曲线，这是后视镜选型最关键的安全指标。

### 8.2 道路布局

与场景 1 相同的 3 车道高速公路。

### 8.3 参考车辆

鲜红色方块（4.5m × 1.85m × 1.5m），高辨识度。分别在左车道和右车道的 5 个距离各放一辆：

| 距离 | Y 位置 | 危险等级 | 透明度 |
|------|--------|---------|--------|
| 10m | -10 | 紧急（必须看到）| 100% 不透明 |
| 20m | -20 | 近距（应能看到）| 83% |
| 30m | -30 | 中距（重要）| 67% |
| 50m | -50 | 远距（有价值）| 50% |
| 70m | -70 | 极远（理想）| 33% 半透明 |

- 左车道 (X = -3.75)：5 辆
- 右车道 (X = +3.75)：5 辆
- 共 10 辆参考车

近处不透明、远处半透明的设计便于在俯视图中快速区分距离。

```
    快车道(-3.75)   本车道(0)   慢车道(+3.75)
         │             │             │
  -10m   ■             │             ■    ← 紧急
  -20m   ■             │             ■    ← 近距
  -30m   ■             │             ■    ← 中距
         │             │             │
  -50m   ▪             │             ▪    ← 远距
         │             │             │
  -70m   ▫             │             ▫    ← 极远

    ■ = 不透明   ▪ = 半透明   ▫ = 浅透明
```

### 8.4 批量渲染模式

除了默认的全量模式（10 辆车同时存在），还支持单车模式——每帧只显示一辆车，避免近处车辆遮挡远处车辆：

```python
from scene_builder import build_lane_change_single
build_lane_change_single('suv', 'L', side='left', distance=20)
```

### 8.5 验证方法

1. 逐距离渲染后视镜视角（共 10 帧：5 距离 × 2 侧）
2. 记录每个距离的参考车是否在镜中可见，以及可见面积占比
3. 绘制"距离-可见性"曲线图
4. 对比不同后视镜类型的检测距离阈值

## 九、使用方法

### 9.1 直接使用预构建文件（推荐）

```
1. 打开 assets/blender-scenes/scene_highway.blend
2. File → Append → 选择车辆/房车/后视镜的 .blend 文件中的物体
3. 调整位置（车辆在原点，房车连接到牵引点，后视镜安装到 A 柱）
4. 设置摄像机到驾驶员眼点位置
5. 渲染
```

### 9.2 通过脚本重新生成

在 Blender Python 控制台中：

```python
import sys
sys.path.insert(0, '/path/to/VMirror-SiM/src')
from scene_builder import build_scenario

# 生成指定场景
build_scenario('highway', 'suv', 'L')       # 高速公路
build_scenario('parking', 'pickup', 'M')     # 停车场
build_scenario('regulatory', 'hatchback', 'L')  # R46 法规（最差情况）
build_scenario('lane_change', 'wagon', 'S')  # 变道检测

# 保存
import bpy
bpy.ops.wm.save_as_mainfile(filepath='/path/to/output.blend')
```

### 9.3 可用组合

| 参数 | 可选值 | 说明 |
|------|--------|------|
| 场景 | `highway`, `parking`, `regulatory`, `lane_change` | 4 种场景 |
| 车辆 | `suv`, `pickup`, `wagon`, `hatchback` | 4 种车型 |
| 房车 | `S`, `M`, `L` | 3 种尺寸 |
