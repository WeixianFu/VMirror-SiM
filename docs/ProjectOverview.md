# 项目概述：VMirror-SiM

## 背景与动机

当车辆拖曳房车（拖挂式旅居车）时，房车车身通常比牵引车更宽，导致标准后视镜无法覆盖车辆后方及侧后方区域，产生显著的视野盲区。这一问题在欧洲拖车文化中尤为普遍——从小型两厢车到皮卡，各类车型都可能拖挂不同尺寸的房车。

VMirror-SiM（Vehicle Mirror Simulation）旨在建立一种**跨平台通用的拖车后视镜视野分析方法论**，系统性地量化不同车型、镜型、房车尺寸组合下的后视镜视野覆盖情况，为后视镜产品设计与法规合规性评估提供仿真数据支撑。

## 技术路线

项目采用 **Blender + Cycles 光线追踪引擎** 作为核心仿真平台：

1. **参数化几何建模**：通过 Blender Python API (bpy) 以参数驱动方式创建简化车辆、房车、后视镜几何体，关键尺寸可配置（见 `config/` 目录下的 YAML 文件）。

2. **AI 辅助 3D 模型生成**：使用 AI 3D 生成工具（Hyper3D 等）基于文本 Prompt 生成低多边形风格化车辆和房车模型，再在 Blender 中校准至真实尺寸。生成 Prompt 及校准要点详见 `docs/VehicleModels.md` 和 `docs/CaravanModels.md`。

3. **真实光线反射仿真**：利用 Cycles 渲染引擎的物理级 Glossy BSDF 材质，模拟后视镜（平面镜 / 凸面镜）的真实反射行为，而非简化的几何光线投射。

4. **标准化测试场景**：搭建包含道路系统、车道标线、地面网格标记、垂直参考柱的测试环境（见 `docs/TestScene.md`），用于量化可视区域边界。

5. **批量仿真矩阵**：遍历车型×镜型×房车尺寸×眼点百分位的参数组合，自动渲染并输出可视范围数据。

## 仿真参数矩阵

| 维度 | 配置项 |
|------|--------|
| 车型 | 皮卡 (Hilux) · SUV (CR-V) · 旅行车 (Passat) · A级小车 (TBD) |
| 后视镜 | 平面镜 · 吸附式凸面镜 · 外套式外壳镜 · 电动升降高端镜 |
| 房车尺寸 | 紧凑型 S · 中型 M · 大型 L |
| 眼点位置 | P5 / P50 / P95（基于 SAE J941 眼椭圆） |

## 坐标系约定

项目统一使用 **Blender 原生坐标系**：

| Blender 轴 | 方向 | 汽车工程含义 | 正方向 |
|-----------|------|------------|--------|
| **X** | 左右 | 车辆横向 | 正 = 副驾侧（右） |
| **Y** | 前后 | 车辆纵向 | 正 = 车头方向 |
| **Z** | 上下 | 车辆垂直 | 正 = 向上 |

原点定义：**前轴中心在地面的投影点 (0, 0, 0)**。

## 法规参考

- **FMVSS 111**（美国）：凸面镜曲率半径 889~1651mm，最小反光面积 ≥126cm²
- **UN R46**（欧洲）：对拖车镜有额外视野覆盖要求
- **SAE J941**：驾驶员眼椭圆标准（P5/P50/P95 百分位）
